# Diffie-Hellman鍵交換 — 公開通信路上での秘密の共有

## 1. はじめに：鍵配送問題

共通鍵暗号（AESなど）は高速で安全だが、通信する双方が**同じ秘密鍵**を共有していなければならない。しかし、まだ安全な通信手段が確立されていない状態で、どうやって秘密鍵を共有するのか。これが**鍵配送問題（Key Distribution Problem）**である。

事前に会って鍵を交換する、あるいは信頼できる第三者を介して配送するといった方法はスケールしない。インターネット上の任意の2者が、事前の秘密共有なしに安全な通信を確立するには、根本的に新しいアプローチが必要であった。

1976年、Whitfield DiffieとMartin Hellmanは画期的な論文 "New Directions in Cryptography" を発表し、**公開通信路上で秘密鍵を共有する**方法を提案した。これが**Diffie-Hellman鍵交換（DH）**であり、現代の暗号通信の基盤となっている。

## 2. 直感的な理解：色の混合のアナロジー

Diffie-Hellman鍵交換は「色の混合」に例えるとわかりやすい。

1. AliceとBobは公開の色（例えば黄色）を共有する
2. Aliceは自分だけの秘密の色（赤）を選び、黄色と混ぜてオレンジを作る
3. Bobは自分だけの秘密の色（青）を選び、黄色と混ぜて緑を作る
4. AliceとBobはそれぞれの混合色（オレンジと緑）を公開通信路で交換する
5. Aliceはオレンジに自分の秘密の色（赤）を混ぜ、Bobは緑に自分の秘密の色（青）を混ぜる
6. 両者は同じ最終色にたどり着く

盗聴者はオレンジと緑を見ることができるが、混合色から元の秘密の色を分離することは困難であり、最終色を導出できない。

## 3. 数学的基盤

### 3.1 離散対数問題

Diffie-Hellman鍵交換の安全性は**離散対数問題（DLP: Discrete Logarithm Problem）**の困難性に基づいている。

素数 $p$ と生成元 $g$ に対して：

$$
\text{Given } g, p, g^a \bmod p, \text{ find } a
$$

$g^a \bmod p$ の計算は（二分累乗法で）効率的だが、$g^a \bmod p$ から $a$ を逆算すること（離散対数の計算）は、$p$ が十分に大きければ計算量的に困難である。

### 3.2 巡回群

素数 $p$ に対して、$\mathbb{Z}_p^* = \{1, 2, \ldots, p-1\}$ は乗法に関して**巡回群**を形成する。位数は $p - 1$ であり、**生成元（原始根）** $g$ が存在する。

生成元 $g$ とは、$g^1, g^2, \ldots, g^{p-1}$（$\bmod p$）がちょうど $\mathbb{Z}_p^*$ の全要素を生成する元のことである。

## 4. プロトコル

### 4.1 基本プロトコル

```
公開パラメータ: 素数 p, 生成元 g

Alice                                   Bob
──────                                  ─────
秘密値 a をランダムに選ぶ                秘密値 b をランダムに選ぶ
A = g^a mod p を計算                    B = g^b mod p を計算

              A = g^a mod p
Alice ─────────────────────────→ Bob

              B = g^b mod p
Alice ←───────────────────────── Bob

s = B^a mod p = g^(ab) mod p          s = A^b mod p = g^(ab) mod p
```

AliceとBobは同じ共有秘密 $s = g^{ab} \bmod p$ を得る。

盗聴者は $p$, $g$, $A = g^a \bmod p$, $B = g^b \bmod p$ を知るが、$a$ や $b$ を知らずに $g^{ab} \bmod p$ を計算することは**計算Diffie-Hellman問題（CDH問題）**と呼ばれ、離散対数問題と同等に困難であると広く信じられている。

### 4.2 正当性

$$
B^a \bmod p = (g^b)^a \bmod p = g^{ba} \bmod p = g^{ab} \bmod p = (g^a)^b \bmod p = A^b \bmod p
$$

べき乗の交換法則 $(g^b)^a = g^{ab} = (g^a)^b$ により、双方が同じ値にたどり着く。

### 4.3 数値例

- $p = 23, \quad g = 5$
- Alice: $a = 6, \quad A = 5^6 \bmod 23 = 15625 \bmod 23 = 8$
- Bob: $b = 15, \quad B = 5^{15} \bmod 23 = 30517578125 \bmod 23 = 19$
- Alice: $s = 19^6 \bmod 23 = 47045881 \bmod 23 = 2$
- Bob: $s = 8^{15} \bmod 23 = 35184372088832 \bmod 23 = 2$

双方が共有秘密 $s = 2$ を得る。

## 5. パラメータの選択

### 5.1 素数 $p$ の選択

$p$ の選択は安全性に直結する。以下の条件が推奨される：

- **ビット長**：最低2048ビット（NIST推奨）。3072ビット以上が望ましい
- **安全素数**：$p = 2q + 1$（$q$ も素数）の形の素数を使用する。これにより、$\mathbb{Z}_p^*$ の位数 $p - 1 = 2q$ が大きな素因数 $q$ を持ち、Pohlig-Hellmanアルゴリズムによる攻撃が無効化される

### 5.2 生成元 $g$ の選択

安全素数 $p = 2q + 1$ を使用する場合、$g$ は位数 $q$ の部分群の生成元として選ぶ。$g = 2$ がよく使われる（$g^q \bmod p \neq 1$ を確認した上で）。

### 5.3 標準パラメータグループ

RFC 7919では、TLS用に事前定義されたDiffie-Hellmanパラメータグループ（ffdhe2048、ffdhe3072、ffdhe4096など）が定義されている。これにより、各実装が独自にパラメータを生成する必要がなくなり、実装の安全性が向上する。

## 6. 中間者攻撃と対策

### 6.1 中間者攻撃（MITM: Man-in-the-Middle Attack）

基本的なDiffie-Hellman鍵交換には**認証メカニズムがない**。攻撃者Malloryが通信路に介入し、AliceとBobそれぞれと個別にDH交換を行うことで、通信内容を傍受・改ざんできる。

```
Alice                    Mallory                    Bob
──────                   ───────                    ─────
a を選ぶ                  m₁, m₂ を選ぶ               b を選ぶ
A = g^a mod p                                       B = g^b mod p

         A
Alice ──────→ Mallory                               Bob
                      M₁ = g^(m₁) mod p
              Mallory ────────────────→ Bob
                                              B
              Mallory ←──────────────── Bob
         M₂ = g^(m₂) mod p
Alice ←────── Mallory

s₁ = M₂^a     s₁ = A^(m₂)   s₂ = B^(m₁)   s₂ = M₁^b
= g^(a·m₂)    = g^(a·m₂)    = g^(b·m₁)     = g^(b·m₁)
```

AliceはMalloryと鍵 $s_1$ を共有し、BobはMalloryと鍵 $s_2$ を共有する。Malloryはすべての通信を復号・再暗号化して中継できる。

### 6.2 対策

中間者攻撃を防ぐには、DH交換に**認証**を組み合わせる必要がある：

1. **デジタル署名**：DHパラメータに対してRSAやECDSAの署名を付与する。TLSではサーバー証明書（PKI）を使ってサーバーの公開鍵を認証する
2. **事前共有鍵（PSK）**：事前に共有した秘密鍵を使って認証する
3. **STS（Station-to-Station）プロトコル**：DH交換と署名を組み合わせた認証付きプロトコル

TLS 1.3では、サーバー証明書によるサーバー認証と、ECDHE鍵交換を組み合わせることで、中間者攻撃を防止しつつ前方秘匿性を実現している。

## 7. Ephemeral DH（DHE/ECDHE）と前方秘匿性

### 7.1 静的DHとEphemeral DH

**静的DH**：長期間同じ秘密鍵 $a$ を使用する。秘密鍵が漏洩した場合、過去のすべての通信が復号される。

**Ephemeral DH（DHE）**：各セッションで新しい秘密鍵を生成する。セッション終了後に秘密鍵は破棄される。

### 7.2 前方秘匿性（Forward Secrecy）

Ephemeral DHが提供する重要な性質が**前方秘匿性（Perfect Forward Secrecy, PFS）**である。

前方秘匿性とは、長期鍵（サーバーの秘密鍵など）が漏洩しても、**過去のセッション鍵が危殆化しない**という性質である。

各セッションのDH秘密鍵はセッション終了後に破棄されるため、攻撃者が長期鍵を入手しても過去の通信を復号できない。

TLS 1.3ではEphemeral DH（ECDHE）が必須であり、すべての接続が前方秘匿性を持つ。これは、大規模な通信傍受を行う国家レベルの攻撃者に対する重要な防御策である。

## 8. 有限体上のDHに対する攻撃

### 8.1 離散対数の計算アルゴリズム

| アルゴリズム | 計算量 | 備考 |
|---|---|---|
| Baby-step Giant-step | $O(\sqrt{p})$ | 汎用アルゴリズム |
| Pollardの $\rho$ 法 | $O(\sqrt{p})$ | メモリ効率的 |
| Pohlig-Hellman | $O(\sqrt{q})$（$q$ は最大素因数） | $p-1$ の因数分解に依存 |
| 数体篩法（NFS） | $L_p[1/3, c]$ | 準指数時間、最良の汎用攻撃 |

### 8.2 Logjam攻撃

2015年に発表された**Logjam攻撃**は、多くのサーバーが512ビットのDHパラメータ（export-grade）を使用していたことを悪用したものである。

さらに重要な発見として、数体篩法の**事前計算**の大部分は素数 $p$ にのみ依存し、特定のDH交換に依存しないことが示された。つまり、同じ $p$ を使う多くのサーバーに対して、一度の大規模な事前計算を行えば、個々のDH交換の解読が高速化される。

1024ビットのDHパラメータに対しても、国家レベルの計算リソースがあれば事前計算が実行可能であるとされ、多くのプロトコルが2048ビット以上への移行を進めた。

### 8.3 小部分群攻撃

攻撃者が $\mathbb{Z}_p^*$ の小さな部分群に属する値を送信し、共有秘密の値域を制限する攻撃。対策として、受信した値が正しい部分群に属することを検証する必要がある。

## 9. ECDHへの移行

### 9.1 楕円曲線上のDiffie-Hellman

**ECDH（Elliptic Curve Diffie-Hellman）**は、有限体上のDHを楕円曲線上で実現したものである。

$$
\text{有限体DH}: s = g^{ab} \bmod p \quad \Longrightarrow \quad \text{ECDH}: S = abG \text{ (on curve)}
$$

ECDHは有限体DHと比較して以下の利点がある：

- **鍵サイズ**：DH-2048（2048ビット）と同等の安全性をECDH-256（256ビット）で実現
- **性能**：鍵生成・鍵共有ともに高速
- **帯域幅**：交換するデータが小さい

### 9.2 X25519

TLS 1.3で推奨される鍵交換方式は**X25519**（Curve25519上のECDH）である。

X25519の特徴：
- 256ビットの鍵長で128ビットの安全性
- 定数時間実装が設計レベルで保証されている
- 不正な入力に対しても安全に動作する
- 非常に高速

### 9.3 TLS 1.3での位置づけ

TLS 1.3では、有限体DHは非推奨となり、ECDHEが標準的な鍵交換方式となっている。サポートされるグループは：

- **X25519**（最も推奨）
- **X448**（より高い安全性）
- **secp256r1**（NIST P-256）
- **secp384r1**（NIST P-384）
- **secp521r1**（NIST P-521）
- **ffdhe2048〜ffdhe8192**（有限体DH、互換性のため）

## 10. ポスト量子暗号との関係

量子コンピュータのShorのアルゴリズムは、有限体上の離散対数問題と楕円曲線上の離散対数問題の両方を多項式時間で解く。これにより、DH/ECDHは量子コンピュータに対して完全に脆弱となる。

現在、以下のポスト量子鍵交換メカニズムが標準化されている：

- **ML-KEM（Kyber）**：格子問題（LWE問題）に基づく鍵カプセル化メカニズム

TLS 1.3のハイブリッド鍵交換（X25519 + ML-KEM-768 のような組み合わせ）が、量子コンピュータ時代への移行戦略として提案・実装されている。Chromeやその他のブラウザでは既にハイブリッド鍵交換の実験的サポートが始まっている。

## 11. まとめ

Diffie-Hellman鍵交換は、現代の暗号通信を可能にした画期的な発明である。事前の秘密共有なしに、公開通信路上で共有秘密を確立できるという発想は、インターネットセキュリティの礎となった。

要点：

- DHの安全性は離散対数問題の困難性に基づいている
- 基本的なDHには認証がなく、中間者攻撃に脆弱。必ずデジタル署名などの認証と組み合わせる
- Ephemeral DH（DHE/ECDHE）を使用して前方秘匿性を確保する
- 有限体DHよりもECDHが効率的であり、X25519が推奨される
- 有限体DHを使う場合は2048ビット以上のパラメータを使用する
- 量子コンピュータ時代に備えてハイブリッド鍵交換への移行を計画する

DiffieとHellmanの1976年の論文は、暗号学の歴史における最も重要な転換点の一つであり、その影響は50年後の今日でもインターネットのあらゆる安全な通信に及んでいる。
