# 楕円曲線暗号（ECC）— なぜ短い鍵で強い安全性を実現できるのか

## 1. はじめに：RSAの限界とECCの登場

RSAが登場した1977年以降、公開鍵暗号はインターネットセキュリティの基盤となった。しかし、RSAには根本的な課題がある。安全性を高めるために鍵長を大きくすると、性能が急激に低下する。RSA-2048で約112ビットの安全性しか得られず、128ビットの安全性にはRSA-3072が必要となる。

**楕円曲線暗号（Elliptic Curve Cryptography, ECC）**は、この問題を解決するために1985年にNeal KoblitzとVictor Millerによって独立に提案された。ECCは楕円曲線上の**離散対数問題**の困難性に基づいており、RSAと比較してはるかに短い鍵で同等の安全性を実現できる。

| 安全性レベル | RSA鍵長 | ECC鍵長 | 倍率 |
|---|---|---|---|
| 80ビット | 1024ビット | 160ビット | 6.4倍 |
| 112ビット | 2048ビット | 224ビット | 9.1倍 |
| 128ビット | 3072ビット | 256ビット | 12倍 |
| 256ビット | 15360ビット | 512ビット | 30倍 |

この効率性の差は、安全性レベルが上がるほど顕著になる。ECCは鍵サイズ、計算速度、帯域幅のすべてにおいてRSAを上回り、現代の暗号システムにおいて主流となりつつある。

## 2. 楕円曲線の数学

### 2.1 楕円曲線の定義

暗号学で使用する楕円曲線は、以下のワイエルシュトラス標準形で定義される：

$$
y^2 = x^3 + ax + b
$$

ここで $4a^3 + 27b^2 \neq 0$（特異点を持たない条件）を満たす必要がある。

実数体上では、この曲線は滑らかな曲線を描く。例えば $y^2 = x^3 - 3x + 5$ のような曲線である。

重要な点として、楕円曲線は楕円とは無関係である。楕円の弧長を計算する楕円積分に関連する方程式から名前が由来しているが、形状は楕円とは全く異なる。

### 2.2 無限遠点

楕円曲線上の点の集合に**無限遠点** $\mathcal{O}$ を加える。無限遠点は、すべての垂直線が「交わる」点として幾何学的に理解できる。無限遠点は後述する群の単位元として機能する。

### 2.3 楕円曲線上の点の加算

楕円曲線上の2点 $P$ と $Q$ の加算 $P + Q = R$ は、以下の幾何学的な規則で定義される：

1. $P$ と $Q$ を通る直線を引く
2. この直線と楕円曲線の第3の交点 $R'$ を求める
3. $R'$ を $x$ 軸に関して反転した点が $R = P + Q$ である

```
         y
         │     R'
         │    ╱•
         │   ╱  ╲
         │  •P   ╲
         │ ╱      •Q
    ─────┼╱────────────── x
         │╲
         │ ╲
         │  •R = P + Q
         │
```

特殊なケース：
- $P = Q$ の場合：$P$ における接線を引き、同じ操作を行う（点の2倍算）
- $P$ と $Q$ の $x$ 座標が同じで $y$ 座標が反対の場合：$P + Q = \mathcal{O}$（無限遠点）
- $P + \mathcal{O} = P$（無限遠点は単位元）

### 2.4 代数的な加算公式

2点 $P = (x_1, y_1)$ と $Q = (x_2, y_2)$ の加算 $R = (x_3, y_3)$ は以下で計算される：

**$P \neq Q$ の場合（点加算）**：

$$
\lambda = \frac{y_2 - y_1}{x_2 - x_1}
$$

**$P = Q$ の場合（点2倍算）**：

$$
\lambda = \frac{3x_1^2 + a}{2y_1}
$$

いずれの場合も：

$$
x_3 = \lambda^2 - x_1 - x_2
$$
$$
y_3 = \lambda(x_1 - x_3) - y_1
$$

### 2.5 楕円曲線群

楕円曲線上の点の集合は、上記の加算演算により**アーベル群**を形成する：

- **閉包性**：2点の加算結果は曲線上の点
- **結合法則**：$(P + Q) + R = P + (Q + R)$
- **単位元**：無限遠点 $\mathcal{O}$
- **逆元**：各点 $P = (x, y)$ に対して $-P = (x, -y)$
- **交換法則**：$P + Q = Q + P$

## 3. 有限体上の楕円曲線

### 3.1 なぜ有限体が必要か

実数体上の楕円曲線は連続的であり、暗号に適さない。暗号学では**有限体**（ガロア体）$\mathbb{F}_p$ 上の楕円曲線を使用する。

素数 $p$ に対する有限体 $\mathbb{F}_p$ は、$\{0, 1, 2, \ldots, p-1\}$ の整数集合に $\bmod p$ の演算を定義したものである。

有限体上の楕円曲線は：

$$
y^2 \equiv x^3 + ax + b \pmod{p}
$$

実数体上の滑らかな曲線とは異なり、有限体上では離散的な点の集合となる。点の数は有限であり、**Hasseの定理**により：

$$
|N - (p + 1)| \leq 2\sqrt{p}
$$

ここで $N$ は曲線上の点の数（無限遠点を含む）である。すなわち、点の数はおおよそ $p$ に近い。

### 3.2 加算公式の有限体版

有限体上では、すべての演算を $\bmod p$ で行う。除算はモジュラ逆元で置き換える：

$$
\lambda = (y_2 - y_1) \cdot (x_2 - x_1)^{-1} \bmod p
$$

### 3.3 スカラー倍算

点 $P$ の**スカラー倍** $kP$ は、$P$ を $k$ 回加算したものである：

$$
kP = \underbrace{P + P + \cdots + P}_{k \text{ 回}}
$$

効率的な計算には**ダブル・アンド・アッド法**（二分累乗法の楕円曲線版）を使用する：

```python
def scalar_multiply(k, P):
    result = O  # point at infinity
    addend = P
    while k > 0:
        if k & 1:
            result = point_add(result, addend)
        addend = point_double(addend)
        k >>= 1
    return result
```

これにより、$O(\log k)$ 回の点演算で $kP$ を計算できる。

## 4. 楕円曲線離散対数問題（ECDLP）

ECCの安全性は**楕円曲線離散対数問題（ECDLP）**の困難性に基づいている。

**問題**：楕円曲線上の点 $P$ と $Q = kP$ が与えられたとき、スカラー $k$ を求めよ。

$kP$ を計算するのは（ダブル・アンド・アッド法で）効率的だが、$Q$ から $k$ を逆算するのは計算量的に極めて困難である。

RSAの素因数分解問題に対しては準指数時間の攻撃（一般数体篩法）が知られているが、適切に選ばれた楕円曲線上のECDLPに対する最良の攻撃は**完全指数時間**（$O(\sqrt{n})$、Pollardの $\rho$ 法）であり、これがECCの効率性の源泉である。

256ビットのECCを破るには約 $2^{128}$ 回の演算が必要であり、これはAES-128と同等の安全性に相当する。

## 5. 標準曲線

### 5.1 NIST曲線

NISTは以下の楕円曲線を標準化している：

| 曲線 | 素数 $p$ のビット長 | 安全性レベル |
|---|---|---|
| P-256 | 256ビット | 128ビット |
| P-384 | 384ビット | 192ビット |
| P-521 | 521ビット | 256ビット |

P-256（secp256r1）が最も広く使われている。TLS、X.509証明書、多くのアプリケーションで標準的に使用される。

NIST曲線に対しては、パラメータの選定過程が不透明であるという批判がある（NSAが関与しており、バックドアの可能性が指摘された）。この懸念は証明されていないが、代替として Curve25519 が広く採用されるようになった。

### 5.2 Curve25519

Daniel J. Bernsteinが設計した楕円曲線である。モンゴメリ曲線の形式を取る：

$$
y^2 = x^3 + 486662x^2 + x \quad \text{over } \mathbb{F}_p, \quad p = 2^{255} - 19
$$

Curve25519の特徴：

- **パラメータの透明性**：すべてのパラメータに明確な設計根拠がある
- **実装の安全性**：定数時間実装が自然に実現できる設計
- **高速性**：特にソフトウェア実装で高いパフォーマンス
- **耐ミス実装性**：不正な入力に対しても安全に動作する

Curve25519上のDiffie-Hellman鍵共有を**X25519**、署名アルゴリズムを**Ed25519**と呼ぶ。

### 5.3 secp256k1

Bitcoinが使用する楕円曲線：

$$
y^2 = x^3 + 7 \quad \text{over } \mathbb{F}_p, \quad p = 2^{256} - 2^{32} - 977
$$

Koblitz曲線の一種であり、効率的な実装が可能である。暗号通貨のエコシステムで広く使われている。

## 6. ECCベースのプロトコル

### 6.1 ECDH（楕円曲線Diffie-Hellman鍵共有）

ECDHは、楕円曲線上でDiffie-Hellman鍵共有を実現するプロトコルである。

```
Alice                                Bob
──────                               ─────
秘密鍵: a (ランダム)                   秘密鍵: b (ランダム)
公開鍵: A = aG                        公開鍵: B = bG

         A = aG
Alice ──────────→ Bob
         B = bG
Alice ←────────── Bob

共有秘密: S = aB = a(bG)             共有秘密: S = bA = b(aG)
         = abG                                 = abG
```

ここで $G$ はベースポイント（生成元）であり、曲線のパラメータとして公開されている。

AliceとBobは同じ共有秘密 $S = abG$ を得る。盗聴者は $A = aG$ と $B = bG$ を知るが、$a$ や $b$ を知ることなく $abG$ を計算することは、**楕円曲線上の計算Diffie-Hellman問題（ECCDH問題）**と呼ばれ、ECDLPと同等に困難であると考えられている。

TLS 1.3では、鍵交換に**ECDHE（Ephemeral ECDH）**が使われる。各セッションで新しい秘密鍵を生成することで、**前方秘匿性（Forward Secrecy）**を実現している。

### 6.2 ECDSA（楕円曲線デジタル署名アルゴリズム）

ECDSAは、DSA（Digital Signature Algorithm）を楕円曲線上で実現したものである。

**鍵生成**：
- 秘密鍵：ランダムな整数 $d$（$1 \leq d < n$、$n$ は曲線の位数）
- 公開鍵：$Q = dG$

**署名生成**（メッセージ $m$ に対して）：

1. ランダムな整数 $k$（$1 \leq k < n$）を生成する
2. $R = kG = (x_1, y_1)$ を計算し、$r = x_1 \bmod n$ とする
3. $e = \text{Hash}(m)$ を計算する
4. $s = k^{-1}(e + rd) \bmod n$ を計算する
5. 署名は $(r, s)$

**署名検証**（メッセージ $m$、署名 $(r, s)$、公開鍵 $Q$）：

1. $e = \text{Hash}(m)$ を計算する
2. $u_1 = es^{-1} \bmod n$、$u_2 = rs^{-1} \bmod n$ を計算する
3. $R' = u_1 G + u_2 Q$ を計算する
4. $R'$ の $x$ 座標が $r$ に等しければ署名は有効

**ECDSAの重要な注意点**：

署名生成時のランダム値 $k$ は絶対に一意でなければならない。同じ $k$ を2つの異なるメッセージに使用すると、秘密鍵 $d$ が漏洩する。

2つのメッセージ $m_1, m_2$ に同じ $k$ を使った場合：

$$
s_1 = k^{-1}(e_1 + rd) \bmod n
$$
$$
s_2 = k^{-1}(e_2 + rd) \bmod n
$$

差を取ると：

$$
s_1 - s_2 = k^{-1}(e_1 - e_2) \bmod n
$$

$$
k = (e_1 - e_2)(s_1 - s_2)^{-1} \bmod n
$$

$k$ が判明すれば：

$$
d = (s_1 k - e_1) r^{-1} \bmod n
$$

2010年にソニーのPlayStation 3のコード署名で、$k$ が固定値であったことが発覚し、秘密鍵が完全に漏洩した（fail0verflow事件）。

この問題を根本的に回避するため、**RFC 6979**では決定論的な $k$ の生成方法が定義されている。メッセージと秘密鍵から決定的に $k$ を導出するため、乱数生成器の品質に依存しない。

### 6.3 EdDSA / Ed25519

**EdDSA（Edwards-curve Digital Signature Algorithm）**は、ツイストエドワーズ曲線上で定義されるデジタル署名アルゴリズムである。Ed25519はCurve25519のツイストエドワーズ形式を使用する。

ECDSAと比較したEdDSAの利点：

- **決定論的署名**：乱数を使わず、秘密鍵とメッセージからのハッシュで $k$ を生成する。ランダム値再利用の脆弱性がない
- **高速性**：エドワーズ曲線の加算公式は統一的であり、条件分岐が少ない
- **耐ミス実装性**：不正な公開鍵や署名に対しても安全
- **バッチ検証**：複数の署名を同時に検証できる

Ed25519はSSH、TLS 1.3、暗号通貨、Signal Protocolなど、幅広いシステムで採用されている。

## 7. ECCの実装上の注意点

### 7.1 サイドチャネル攻撃

ECCの実装では、以下のサイドチャネル攻撃に注意が必要である：

- **タイミング攻撃**：スカラー倍算の処理時間からスカラー値（秘密鍵）が推測される
- **電力解析攻撃**：消費電力パターンからスカラー倍算の各ステップが推測される
- **キャッシュ攻撃**：メモリアクセスパターンから情報が漏洩する

対策：
- **モンゴメリラダー**：入力に依存しない一定のパターンで点演算を行う
- **座標のランダム化**：射影座標にランダムな倍率を乗じる
- **定数時間演算**：条件分岐や入力依存のメモリアクセスを排除する

Curve25519はこれらの対策が設計レベルで組み込まれており、安全な実装が容易である。

### 7.2 無効曲線攻撃

攻撃者が不正な点（元の曲線上にない点）を送信した場合、弱い群上で計算が行われ、秘密鍵が漏洩する可能性がある。

対策として、受信した点が正しい曲線上にあることを**必ず検証**する必要がある。X25519はプロトコルの設計上、この検証が不要になっている（全入力が有効な点にマッピングされる）。

### 7.3 乱数生成の重要性

ECDSAの署名における $k$ の生成には、暗号学的に安全な乱数生成器が不可欠である。乱数の品質が低い場合、秘密鍵が漏洩するリスクがある。

2013年、Androidの `SecureRandom` の実装に欠陥があり、Bitcoinウォレットから秘密鍵が漏洩する事件が発生した。

## 8. 楕円曲線暗号の将来

### 8.1 量子コンピュータの脅威

ECCはRSAと同様に、量子コンピュータのShorのアルゴリズムによって破られる。ECDLPに対するShorのアルゴリズムの計算量は $O(n^3)$（$n$ はビット長）であり、256ビットのECC鍵は数千量子ビットの量子コンピュータで破られると推定されている。

ポスト量子暗号（格子暗号、符号暗号など）への移行が進んでいるが、ECCは量子コンピュータが実用化されるまでの間、引き続き主流の公開鍵暗号として使われると予想される。

### 8.2 同種写像暗号

楕円曲線間の**同種写像（isogeny）**に基づく暗号方式が研究されていた（SIKE/SIDHなど）。ポスト量子暗号の候補として注目されていたが、2022年にCastryckとDecruによって多項式時間の攻撃が発見され、SIKEは破られた。これは暗号学のコミュニティにおいて大きな衝撃であり、新しい暗号方式の安全性評価の重要性を改めて示す出来事であった。

## 9. まとめ

楕円曲線暗号は、数学的に美しい代数構造の上に構築された、効率性と安全性を両立する暗号技術である。RSAと比較して以下の優位性がある：

- **鍵サイズ**：同等の安全性に対して大幅に短い鍵
- **性能**：署名生成・検証、鍵共有のいずれも高速
- **帯域幅**：小さな鍵・署名サイズによる通信量の削減

現代のベストプラクティス：

- 鍵共有には**X25519**（またはX448）を使用する
- 署名には**Ed25519**（またはEd448）を使用する
- NIST曲線が必要な場合はP-256を使用する
- ECDSAを使う場合はRFC 6979の決定論的 $k$ 生成を使用する
- 長期的にはポスト量子暗号との併用（ハイブリッドモード）を検討する

ECCは「より少ないリソースでより高い安全性を」という暗号学の理想を体現した技術であり、RSAからの世代交代を確実に進めている。
